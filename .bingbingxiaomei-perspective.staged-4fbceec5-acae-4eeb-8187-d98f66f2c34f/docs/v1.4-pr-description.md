## v1.4: 容错与清晰度双维提升 (dim3+dim5)

### 背景

Darwin v2.0 基线评估 v1.3 总分 77.4，dim5(清晰度)=6 和 dim3(容错)=7 为双短板。v1.4 针对两维度做 5 处改动，总分 +1.8 到 79.2。

### 改动 (6 处)

**dim5 清晰度 (M1+M2)**:

- **M1 沉默规则改写** — `≥4轮 / 对话轮次计数` → `≥3轮 / 关键词累计计数`。原"对话轮次计数"LLM 无法执行（无跨轮记忆），改为"同主题词重复 ≥3 次"可在当前轮检测。沉默规则 3 行结构全部保留（港股/美股/加密货币仍为立即拒答，质量闸"无法提取 ≥2 心智模型"保留）。
- **M2 Step 1.B 路由决策树** — 新增 6 类问题路由（沉默→个股→技术信念张力→行业/板块→宏观/政策→纯框架），按优先级排列，替代原有 LLM 隐式自行判断。第 3 类"技术-信念张力"为新增类型（原 5 类缺失），对应现存 line 67 显式声明。插入位置在 Step 1 判断原则与 Step 1.5 之间。

**dim3 容错 (3.1+3.2+3.3)**:

- **3.1 认知/行为失败处理表** — 5 场景 if-then fallback：沉默规则误触发（二确流程）、角色语气泄漏（全文替换）、错误模型选择（回退模型 1）、双重身份混乱（统一替换）、元层面自指（立即拒绝）。与已有工具失败处理表（5 场景）形成 A/B 双表结构。
- **3.2 表达 DNA 漂移检测** — 角色内持续软规则（2 问：小美我 ≥1 次 / 无单独"我"指代自身），不输出自检行，仅 LLM 内部检查。与退出角色后的残留检测（3 条）功能正交——一处是角色内约束，一处是退出后清理。
- **3.3 双表统一** — `### 工具调用失败处理` → `### A. 工具调用失败处理`，`### B. 认知/行为失败处理`。两表相邻，结构对称。

**测试**:

- test-prompts 16→20：case 17（港股沉默）、18（技术-信念路由）、19（纯框架无足迹）、20（角色语气泄漏）
- case 8 单引号 JSON bug 修复（v1.2 polish 遗留，`json.load()` 失败）
